## Сценарій відкриття задачі
- студент копіює назу задачі
- судент надаає команду LOGIN і вводить назву задачі
- б1 відправляе логін і пароль 
- Псет повертає токен доступу
- б1 складає список встановлених ШІ-розширень
- б1 відправляє назву задачі і складений список ШІ-розширень
- Псет створює новий або знаходить старий тікет
- Псет в перший запис нового тікету в поле code заность список ШІ-розширень
       в черговий запис старого тікету в поле code пише слово SECONDHAND
- б1 створює файл вирішення, зберігає його в теці і відкриває вікно редагування

## Сценарій перевірки
- студент набирає код, а б1 виконує трасування
- студент надає команду CHECK
- б1 вирізає частину між BEGIN...END і разом з накопиченою трасою відсилає дл Псет
- Псет додає накопичену трасу до вже збереженої 
- Псет відсилає код на перевірку до служби і отримуе результат
- Псет пересилає результат перевірки до б1
- б1 очищає трасу і показує результат студенту
